2019-08-29
技术

利用hexo工具编写个人博客

  1. 安装
    打开Git Bash，输入以下命令安装Hexo
    npm install -g hexo-cli

  2. 初始化
    打开Git Bash，输入命令
    hexo init [folder]
    [folder]代表的是你的hexo文件夹名，如果不写，就默认在当前文件夹初始化。

  3. 然后进入你的hexo文件夹
    cd [folder]

  4. 生成静态文件
    需要执行generate命令生成静态文件，才能部署到服务器
    hexo g

安装应用
adb install -r com.android.sample.apk

卸载应用
adb uninstall com.android.sample

清除应用缓存
adb shell pm clear com.android.sample

启动app 并且启动打印时间
adb shell am start -W -n com.android.sample.MainActivity

停止应用
adb shell am force-stop com.jingdong.sample.MainActvity

抓取 log
adb logcat -v time >> log.txt

2018-02-26
技术

导出导入数据

  1. 导出所有数据库
    该命令会导出包括系统数据库在内的所有数据库
    mysqldump -uroot -proot --all-databases >all.sql

  2. 导出db1, db2两个数据库的所有数据
    mysqldump -uroot -proot --databases db1 db2 >twodbs.sql

  3. 导出db1中的a1、a2表
    mysqldump -uroot -proot --databases db1 --tables a1 a2 >twotables.sql

  4. 导出db1库中表a1中id=1的数据
    mysqldump -uroot -proot --databases db1 --tables a1 --where='id=1'

  5. 只导出表结构不导出数据
    mysqldump -uroot -proot --no-data --databases db1 >my.sql

  6. 跨服务器导出导入数据,-C启用压缩传送
    mysqldump --host=h1 -uroot -proot -C --databases db1 |mysql --host=h2 -uroot -proot db2

搭建java web工程中遇到的问题

2018-02-12
技术

问题

 前一段时间，我第一次尝试进行Java Web开发。需要将原来工程中的旧es(Elasticsearch)
服务相关的代码，替换成使用JDBC template方式。一开始，我直接在工程中进行开发，参考同事发给我的demo。我将其它涉及到的3个library引入到工程的pom.xml文件中。这个工程本地不能运行，需要发布到线上环境。我这边将工程打包成war文件，部署到线上tomcat容器下，结果运行发现，查询数据的接口没有响应，也没有错误日志打印。我排查了es相关的配置文件，发现配置文件没有问题。

排查过程

1. 为了排查此问题，我重新创建了一个Test工程。写了一个简单的测试接口，也在此工程中的pom.xml文件中，
   引入了那三个库，及原来的es库。并且复制es配置文件到此工程。最后打成war包，部署到线上环境，发现
   还是一样的问题，接口没有响应。

2. 通过上面的测试，我肯定工程中的逻辑代码应该没有问题，问题可能出现在jar包的引入方面。我通过与同事
   发给我的demo工程相比较，发现我的工程中的es的版本比较低。我替换成demo工程中的es版本，重新发布上线，
   发现接口可以走通了。
